Lab 4 - Pseudocoding and Problem-Solving

Challenge

The biggest challenge I ran into was figuring out how to write the pseudocode for how the game detects what player won. Rock, Papers, and Scissors has a lot of win conditions so it made it hard for me to figure out how the code could calculate those results. I ended up writing condition statements as placeholders for the game.

Problems

I didn't really have a lot of problems with this lab assignment. The only actual problem I had was figuring out how to do sub tasks in the form as sub bullet points on the lab website for the every day task implementation. I later figured out by experimenting that making an unordered list with a "li" tag inside made it a bullet point instead of a number in an ordered list.
